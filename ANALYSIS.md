# Анализ: FPDF-подобная библиотека на Nim

## Что уже есть

В Nim уже существует **nimPDF** (jangko/nimpdf) — библиотека, вдохновлённая именно PHP FPDF. Однако она:
- Не обновлялась активно
- Не реализует часть важных фич (аннотации, гиперссылки, outline, цифровые подписи)
- Имеет свой собственный API, отличный от FPDF

## Масштаб задачи

| Уровень | Что входит | Объём кода | Срок (1 чел.) |
|---------|-----------|-----------|---------------|
| **MVP (= PHP FPDF)** | Страницы, текст (Cell/MultiCell/Write), 5 встроенных шрифтов, линии/прямоугольники, JPEG/PNG, ссылки, метаданные | ~2 000 строк | 2–4 недели |
| **Средний (PHP + часть Python)** | + Unicode/TTF subsetting, + больше графических примитивов (круг, эллипс, полигон), + таблицы, + сжатие | ~8 000–10 000 строк | 2–3 месяца |
| **Полный (= fpdf2 Python)** | + HTML→PDF, + SVG, + шифрование, + BiDi/RTL, + Tagged PDF, + цифровые подписи | ~25 000–30 000 строк | 6–12 месяцев |

## Основные блоки работы

### 1. Ядро PDF (1–2 недели) — самое простое
- Генерация PDF-структуры (объекты, xref-таблица, trailer)
- Бинарный вывод через `std/streams`
- Управление страницами, позицией, полями

### 2. Текст и шрифты (2–4 недели) — средняя сложность
- 14 базовых шрифтов PDF (метрики, кодировки) — просто
- TTF парсинг и subset embedding — **сложно**, это самая трудоёмкая часть
- Unicode / UTF-8 — `std/unicode` поможет, но маппинг glyph→CID нетривиален

### 3. Графика (1 неделя)
- Линии, прямоугольники, кривые Безье — прямое отображение на PDF-операторы
- Цвета (RGB, CMYK) — просто

### 4. Изображения (1–2 недели)
- JPEG — можно встраивать as-is в PDF (самое простое)
- PNG — нужно декодировать (nimPNG поможет), обработать альфа-канал
- Кеширование, дедупликация

### 5. Сжатие (пара дней)
- Deflate через `zippy` (pure Nim) или `zip/zlib` — тривиально

### 6. Продвинутые фичи (если нужны, по 1–3 недели каждая)
- Шифрование (AES/RC4) — средняя сложность
- HTML→PDF парсер — сложно
- SVG рендерер — **очень сложно** (~2000 строк в fpdf2)
- BiDi/RTL текст — сложно (UAX #9 алгоритм)

## Что упрощает задачу на Nim

- **Компиляция в C** — нативная производительность, лёгкий FFI
- **`std/streams`** — бинарный I/O из коробки
- **`std/unicode`** — UTF-8 поддержка
- **zippy** — pure Nim deflate
- **nimPNG** — PNG декодирование
- Можно подглядывать в nimPDF как референс

## Что усложняет

- **TTF subsetting** — самая сложная часть, ~3500 строк в fpdf2
- **Спецификация PDF** — документ на 750 страниц (ISO 32000)
- **Тестирование** — нужно визуально проверять результат, автоматизация сложна
- **Маленькая экосистема Nim** — меньше готовых решений

## Референсные библиотеки

### FPDF PHP (оригинал)
- 1 файл, ~1 934 строки, 48 публичных методов
- 0 зависимостей
- Минималистичная архитектура: один монолитный класс

### fpdf2 Python (расширенная версия)
- 35 модулей, ~27 700 строк, 80+ публичных методов
- 3 зависимости (Pillow, defusedxml, fontTools)
- Модульная архитектура с разделением ответственности

### nimPDF (существующая Nim библиотека)
- Вдохновлена PHP FPDF
- Pure Nim, MIT лицензия
- Поддерживает: PNG, JPEG, BMP, TTF/TTC subsetting, шифрование (ARC4, AES)
- Не реализовано: аннотации, гиперссылки, outline, цифровые подписи

## Рекомендуемая стратегия

1. **Начать с MVP уровня PHP FPDF** (~2000 строк, 48 методов) — реально за 2–4 недели
2. Использовать PHP FPDF как основной референс (1 файл, чистая логика, легко портируется)
3. Добавлять фичи из fpdf2 итеративно по мере необходимости
4. TTF subsetting отложить — начать с 14 базовых шрифтов
